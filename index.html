B 1 - Contexto y fundamentos

T 1, 2 y 3 - Definición de interfaces, elementos, resoluciones y unidades

1 - DEFINICIÓN DE INTERFACES Y UX

INTERFACES WEB:
Son sistemas gráficos que permiten la interacción humano-máquina mediante tecnologías web (HTML, CSS y JavaScript).
Un formulario de login es una interfaz que recibe datos del usuario.

UX (Experiencia de Usuario):
No solo es el diseño visual, sino todo el proceso de interacción del usuario con el producto.

  • Usabilidad: que sea fácil de usar.
  • Accesibilidad: que funcione para todos.
  • Emociones: que la experiencia sea satisfactoria.

2 - ELEMENTOS DE LAS INTERFACES

INTERACTIVOS:

  • Entrada de datos: campos de texto (<input>), checkboxes (input type='checkbox')
  • Navegación: menús (<nav>), enlaces (<a href=' '>)
  • Contenedores: acordeones, pestañas (se usan <div> + JavaScript)

GRÁFICOS:

  • Color: Psicología.
  • Tipografía: Legibilidad.

3 - RESOLUCIONES Y UNIDADES

RESOLUCIONES:

  • Móvil vs Escritorio: En móviles, el CSS Pixel Ratio hace que un píxel lógico no equivalga a uno físico.
  • Breakpoints: Puntos donde el diseño se adapta (@media (max-width: 768px) para tablets).

UNIDADES:

  • Absolutas: px (fijos).
  • Relativas:
    • %: relativo al contenedor padre.
    • rem: relativo al tamaño de fuente del html.
    • vh/vw: relativo al tamaño de la pantalla.

B2 - Guías de estilo y prototipado

4 - PROTOTIPADO

Un prototipo es un modelo interactivo o estático que simula cómo funcionará una interfaz web o app antes de desarrollarla. No es el producto final, sino una representación que ayuda a validar ideas.

Objetivos principales:

  • Validar la usabilidad.
  • Testear flujos.
  • Comunicar ideas (al equipo y clientes).
  • Ahorrar costes: corregir errores en esta fase es más barato que después de programar.

Fases del proceso de diseño:

  • Investigación inicial → Wireframes (baja fidelidad).
  • Diseño visual → Mockups (alta fidelidad).
  • Validación → Prototipo interactivo (figma/Adobe XD).
  • Desarrollo → Se programa basándose en el prototipo aprobado.

Ventajas:

  • Ahorrar tiempo y dinero.
  • Mejora la usabilidad con feedback real.
  • Facilita la colaboración en equipo.

Desventajas:

  • Puede generar falsas expectativas si no se explica que es un modelo no funcional.
  • Herramientas complejas que requieren aprendizaje.
  • Un prototipo muy detallado puede demorar el proceso.

Partes de un prototipo:

  • Sitemap: mapa jerárquico de las páginas.
  • Wireframe: esquema sin diseño visual.
  • Mockup (High Fidelity): diseño visual con colores, tipografías e imágenes.
  • Prototipo interactivo: simula la navegación clickeable.

B3 - Implementación técnica

5 - Estructura del proyecto


<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>home</title>
  <link rel="stylesheet" href="CSS/Style.css">
  <script src="JS/Scripts.js"></script>
</head>
  • <meta charset="UTF-8">: Define la codificación de caracteres.
  • <meta name="viewport">: Asegura que la web se adapte a dispositivos móviles (responsive design).

6 - Estructura básica y navegación


<header id="encabezado">
  <nav id="navbar">
    <ul id="menu">
      <li class="elementoMenu"><a href="#novedades">Novedades</a></li>
    </ul>
  </nav>
</header>

<a href="#novedades" onClick="marcarPestana('collares', 'tabCollares')">Collares</a>
  • <header>: Contenido introductorio.
  • <nav>: Define la barra de navegación.
  • <ul>: Lista no ordenada, estructura estándar para menús.
  • href="#novedades": Enlace interno a una sección con id="novedades".
  • onClick: Evento JavaScript para cambiar pestañas dinámicamente.

7 - Aplicación de estilos CSS


.elementoMenu > a {
  color: #FFFFDD;
  transition: color 0.5s;
}
.elementoMenu > a:hover {
  color: #87CFFF;
}
  • Selector >: Aplica solo a hijos directos (<a> dentro de .elementoMenu).
  • Transition: Suaviza el cambio de color al hacer hover.

8 - Maquetación, columnas y menús


#tituloEncabezado {
  display: flex;
  justify-content: center;
  align-items: center;
}

#submenu {
  display: none;
  position: absolute;
}
.elementoMenu:hover #submenu {
  display: block;
}
  • display: flex → crea un contenedor flexible.
  • justify-content y align-items: centran el contenido horizontal y verticalmente.
  • display: none → oculta el submenu inicialmente.
  • position: absolute → posiciona el submenú respecto al elemento padre.

9 - Estilos de texto


<link href="https://fonts.googleapis.com/css2?family=Raleway&display=swap" rel="stylesheet">

.tituloSeccion {
  font-family: "Raleway", sans-serif;
  text-shadow: 0px 1px 2px rgba(0, 0, 0, 0.75);

  • text-shadow: añade sombra al texto para mejorar la legibilidad.
